# CODEX_TO_CLAUDE - Solar Project Codex Handoff

Claude가 Codex 태양계 공개본을 리뷰할 때 이 파일을 먼저 읽으면 된다.

## 최신 상태

- 기준 날짜: 2026-05-24 KST
- 공개 앱: `solar-project-codex/index.html`
- 사건의 지평선 직접 확인 URL: `solar-project-codex/index.html?focus=event-horizon`
- 진행 단계: Step 8
- Codex 구현 커밋: `8844253 Enhance event horizon black hole visualization`
- 핵심 변경: 포커스 메뉴에 `태양계 전체`와 `사건의 지평선` 추가, 사건의 지평선 시각화 보강

## Step 7 구현 내용

- 포커스 드롭다운 최상단에 `태양계 전체`를 추가했다.
- 정보 패널의 `전체 보기` 버튼은 이제 태양이 아니라 `태양계 전체` 프레임으로 돌아간다.
- `사건의 지평선` 항목은 Sagittarius A* 사건의 지평선 교육용 시각화로 구현했다.
- 사건의 지평선 시각화는 기본 숨김이며, `사건의 지평선`을 선택했을 때만 표시된다.
- 사건의 지평선 포커스는 실제 거리 `26,670 ly`, 실제 반지름 `0.085 AU`를 정보 패널에 표시한다.
- 실제 사건의 지평선은 너무 작으므로 장면에서는 관찰 가능한 크기로 확대한 블랙홀 경계/포톤링 스타일 메시를 사용한다.

## Step 8 구현 내용

- 사건의 지평선 시각화를 단순 경계 표시에서 블랙홀 장면으로 바꿨다.
- 절차적 accretion disk 텍스처, 중심 암흑 코어 마스크, 렌즈 링, 외곽 헤일로, 디스크 입자, 약한 극방향 제트를 추가했다.
- 포커스 카메라는 성간 거리 비율이 아니라 블랙홀 시각 반지름 기준으로 접근해 화면에서 실제로 검사 가능한 크기로 보이게 했다.
- 모든 블랙홀 레이어는 기존 조건부 표시 규칙을 유지해 `사건의 지평선`을 선택했을 때만 켜진다.
- `?focus=event-horizon` URL 파라미터로 사건의 지평선 포커스를 바로 열 수 있다.

## 유지해야 할 동작

- 카이퍼 벨트와 오르트 구름은 기본 화면에서 계속 숨김이어야 한다.
- `태양계 전체`를 선택해도 카이퍼/오르트 경계가 자동으로 켜지면 안 된다.
- `사건의 지평선`을 선택했을 때만 사건의 지평선 메시와 가이드 라인이 보여야 한다.
- 부산 조석표와 실시간 달 위상 위젯은 기본 부산 설정을 유지한다.

## 검증 포인트

- 포커스 메뉴: `태양계 전체`, `사건의 지평선` 항목 존재
- `태양계 전체`: 정보 패널 제목이 `태양계 전체`이고 행성계 전체 카메라 프레임으로 이동
- `Event Horizon`: 정보 패널 제목이 `Sagittarius A* Event Horizon`, 거리 단위가 `ly`, 반지름 단위가 `AU / km`
- `사건의 지평선`: accretion disk, 렌즈 링, 헤일로, 제트가 화면에서 식별 가능
- Playwright 검증은 desktop/mobile에서 nonblank WebGL, 조건부 표시, 정보 패널 값을 확인한다.
